Module 03

Connectivity & Paths

Now that we have defined graphs, we must understand how to move through them. This module covers the fundamental concepts of navigation: walks, trails, paths, and how we determine if a network is "connected."

Walks, Trails, and Paths

In standard language, we use "path" and "walk" interchangeably. In Graph Theory, the distinction is crucial.

Definition 3.1: The Hierarchy
  1. Walk: A sequence of vertices and edges (vertices and edges can be repeated).
  2. Trail: A walk with no repeated edges.
  3. Path: A walk with no repeated vertices (which implies no repeated edges).
Figure 3.1: A walk that visits node 1 twice (not a path).

Connectivity

A graph is connected if there is a path between every pair of vertices. If the graph is broken into disjoint pieces, those pieces are called Connected Components.

Bridges (Cut Edges)

Some edges are more critical than others. A Bridge (or cut-edge) is an edge whose removal increases the number of connected components.

Theorem 3.1 An edge \( e \) is a bridge if and only if it does not lie on any cycle.
